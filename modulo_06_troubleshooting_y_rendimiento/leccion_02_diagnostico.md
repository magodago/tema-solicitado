# Diagnostico

## Objetivos
- Comprender los pasos básicos para diagnosticar problemas de rendimiento en sistemas informáticos.
- Identificar las herramientas y métodos más efectivos para realizar un diagnóstico preciso.

## Contenido
El diagnóstico es una parte crucial del troubleshooting, ya que permite identificar rápidamente la causa raíz de los problemas de rendimiento. Para hacerlo eficazmente, se debe seguir un proceso sistemático que incluye observar el comportamiento del sistema, recopilar datos relevantes y analizarlos para determinar las posibles causas. Los primeros pasos implican revisar registros de errores, monitorear el uso de recursos del sistema (CPU, memoria, disco) y verificar la configuración actual. Es importante utilizar herramientas específicas como Task Manager en Windows o top en sistemas Linux para obtener información precisa sobre el rendimiento.

## Ejercicio
Realice un análisis preliminar de rendimiento en su computadora personal siguiendo estos pasos:
1. Abra el Task Manager (Windows) o la terminal y ejecute `top` (Linux).
2. Observe los procesos que consumen más recursos.
3. Revise los registros de eventos del sistema para cualquier mensaje de error relacionado con estos procesos.
4. Realice un resumen breve de sus observaciones y posibles causas.

## Resumen
- El diagnóstico implica una serie de pasos metodológicos para identificar problemas de rendimiento.
- Es fundamental utilizar herramientas específicas para recopilar datos precisos sobre el uso de recursos del sistema.
